🔐  Week 11 – Security: CSRF & XSS Protection


CSRF Protection

Enable csrf filter in app/Config/Filters.php

Add <?= csrf_field() ?> to every POST form

Demonstrate a 403 Forbidden response when the token is absent

Explain in documentation how the token prevents forged requests


XSS Prevention

Use esc($variable, 'html') on all user-controlled output

Demonstrate stored XSS blocked: type <script>alert(1)</script>

Enable XSS filter globally in Filters.php

Include before/after screenshots: raw vs. escaped output

📦  Deliverable:
A working CI4 form protected against CSRF and XSS. Include a written security report (min. 300 words) explaining each attack type and the countermeasure used.

⚙️  Weeks 12–13 – Advanced CI4 Features


📁 File Uploads

Build an image/file upload form (enctype multipart)

Use UploadedFile class: validate type, size, MIME

Store files securely in writable/uploads/

Display uploaded file on the next page


📧 Email

Configure SMTP settings in app/Config/Email.php

Send an HTML email on a user action (e.g. registration)

Include plain-text fallback message

Log email send status in writable/logs/


📄 Pagination

Display a paginated data listing (5+ records per page)

Integrate search query with pagination (maintain params)

Use $model->paginate(5) and $pager->links()

Show current page and total record count


⚡ Performance

Enable and demo CI4 Debug Toolbar in dev mode

Implement page caching on at least one heavy route

Optimize at least one query (use select() / indexes)

Include toolbar screenshot with SQL query count

📦  Deliverable:  Working CI4 application demonstrating all four feature areas — submit controller, model, view files, and a screenshot of each working feature.

🧪  Week 14 – Unit Testing & Debugging


PHPUnit Test Suite

Create at least 3 test methods in tests/app/

Extend CIUnitTestCase; file must end in Test.php

Cover: testHomePage() → assertStatus(200)

Test a model method with assertEquals()

Test a validation rule passes/fails

Run suite via ./vendor/bin/phpunit; screenshot PASS output


Debugging Skills

Use dd() to inspect a model or request object

Read & explain a stack trace (screenshot + annotation)

Fix at least one real bug found using these tools

Document: what was wrong & how you found it

Required Assertions

🚀  Week 15 – Deployment & Cloud Hosting


01
Production Config

Set CI_ENVIRONMENT = production in .env

Configure baseURL in App.php

Disable Debug Toolbar

Remove dev routes and test data


02
File Transfer

Deploy via SFTP (FileZilla/WinSCP) OR git push

Set writable/ folder permissions to 755

Configure .htaccess for URL rewriting

Redirect all traffic to /public/


03
Database Migration

Export DB via mysqldump or phpMyAdmin

Import .sql to production host

Update .env with live DB credentials

Verify all tables migrated correctly


04
Security & SSL

Obtain SSL certificate (Let's Encrypt / host)

Add .env to .gitignore — never commit secrets

Disable directory listing (Options -Indexes)

Strong unique DB password per environment

📦  Deliverable:  Submit a live URL + a 1-page deployment log documenting each step, issues encountered, and how they were resolved.
